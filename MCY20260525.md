# Manual Test Checklist — 2026-05-25

Covers the manual steps deferred from the last two merged PRs:

- **#13** — `fix(extensions): make app embeds discoverable + move asset portal to Customer Account UI`
- **#14** — `feat(phase-1i): merchant onboarding wizard + magic-link welcome`

Steps are ordered so the test buyer created by the onboarding wizard (§3)
becomes the approved B2B buyer used to verify the asset portal (§5). Don't
re-run a section unless something earlier in the chain changed.

---

## 1. One-time setup

- [ ] Dev store available with the app installable (or already installed —
      if so, uninstall first so the onboarding banner is reachable from a
      clean state).
- [ ] Local checkout on `main` with the merged PR #13 + #14 commits
      (`a677a54` or later).
- [ ] `pnpm install` clean; `shopify` CLI logged in to the dev-store
      partner org.
- [ ] At least one product on the dev store with non-trivial inventory
      (any product works — used for the asset portal visibility check).
- [ ] Inbox access to the `test-buyer+<shop-handle>@<EMAIL_FROM-domain>`
      catch-all (per DECISIONS #15) so magic-link emails are visible.

## 2. Deploy & install

- [ ] `shopify app deploy` from the repo root. Confirm the CLI lists
      the following extensions in the deploy summary:
  - `theme-app-extension` (theme app extension)
  - `customer-account-asset-portal` (customer-account UI extension)
  - the three Functions (cart-transform, cart-validation, delivery-customization)
- [ ] Install the app on the dev store via the partner-dashboard install
      link. OAuth completes; admin lands on the home page.

**Pass criteria:** Deploy summary mentions both extensions above. Install
round-trip finishes without scope-grant errors.

## 3. Onboarding wizard (PR #14)

- [ ] On admin home, a *Continue setup* banner is visible. The Onboarding
      nav-card shows "Live" with a progress count (initially `0 / 6`).
- [ ] Click into `/onboarding`. The wizard renders six step-cards with a
      progress bar at the top.
- [ ] **Step 1 — Detect:** click *Detect existing setup*. Counts populate
      for companies / catalogs / markets / wholesale-tagged customers
      (catalogs may show "unknown" on older API versions — that's fine).
      Step badge flips to ✓.
- [ ] **Step 3 — Tiers:** click the link-out to `/tiers`, create one tier
      (e.g. *Silver*, 10% off), return to the wizard, click *Mark done*.
- [ ] **Step 4 — Application form / approval mode:** link-out to
      `/settings`, save the form (any non-empty change is fine), return,
      *Mark done*.
- [ ] **Step 5 — Asset library:** click *Skip* (the only skippable step).
      Badge flips to *Skipped*.
- [ ] **Step 6 — Test buyer:** click *Create test buyer*. The card shows
      the created email (must be `test-buyer+<shop-handle>@<EMAIL_FROM-domain>`)
      and a Shopify customer GID. Click *Resend magic link* — confirms a
      second invite mutation fires without error.
- [ ] **Step 7 — Go-live checklist:** review the three storefront-enable
      reminders, *Mark done*.
- [ ] Progress bar reads `6 / 6`; the home-page banner disappears; status
      flips to *completed*.
- [ ] (Optional, only if D1 console handy) In D1, `select settings_json
      from shops where ...` — `onboarding.status` is `completed`, each
      step has a populated `data` or `completed_at`.

**Email check:**

- [ ] The magic-link email from Step 6 lands in the `test-buyer+@`
      catch-all. Subject and sender are from the Shopify-side
      `customerSendAccountInviteEmail` template (not from Resend).
- [ ] Clicking the link logs the test buyer into the customer account on
      the storefront.

**Pass criteria:** Wizard reaches `6 / 6`. Test buyer Company is visible
in Shopify admin → Customers → Companies. Magic-link email arrived and
the link works.

## 5. Theme app embeds — discoverability (PR #13)

- [ ] In the dev store, open the theme editor on the active theme.
- [ ] Open the **App embeds** panel (gear icon in the left rail).
- [ ] Two entries from this app are listed:
  - **B2B Tier Price**
  - **B2B Wholesale Application**
- [ ] Enable **B2B Tier Price** globally; save. Open a PDP on the
      storefront as a guest — no console errors, no FOUC, no visible
      change for guests (it no-ops off-PDP and for non-B2B customers).
- [ ] Enable **B2B Wholesale Application**, set the `apply_path` setting
      to the path of the page you embed the form on (e.g. `/pages/apply`),
      save. Visit that path on the storefront — the application form

**Pass criteria:** Both entries appear in the *App embeds* panel (not
under *Add block* in a section — that's the pre-PR-#13 behaviour and
would indicate the deploy didn't take). `apply_path` gating works.

## 6. Customer Account asset portal (PR #13)

Uses the test buyer from §3. If §3 was skipped, log in as any approved
B2B buyer instead.

- [ ] In Shopify admin → Customer Accounts → UI extensions, enable the
      `customer-account-asset-portal` extension. Set its `worker_base_url`
      setting to the deployed Worker host (the same host that serves
      `/admin/*` for this app).
- [ ] Upload at least one asset in admin → `/assets` and set its
      visibility to *all B2B* (or to the test buyer's tier / company).
- [ ] Open the storefront, click the magic link from §3 (or sign in as
      the buyer), navigate to *Account*.
- [ ] On the order index, a card / block links through to *Asset portal*.
- [ ] The asset-portal full-page extension loads, lists the asset you
      uploaded, and the download link works (file streams down via the
      Worker, not via a public R2 URL).
- [ ] Sanity: as a buyer **not** in the asset's visibility scope (a
      different tier, or sign out), the asset does not appear in the
      list and a direct download URL returns 403/404.

**Pass criteria:** The portal page is reachable from the customer
account nav, visibility filtering matches what the admin set, and
downloads stream through the Worker.

## Quick triage matrix

| Symptom | Likely cause | Where to look |
|---|---|---|
| App embeds don't show up in theme editor | `shopify app deploy` didn't run, or wrong theme | `extensions/theme-app-extension/shopify.extension.toml` |
| Customer-account asset portal page absent | extension not enabled in admin → Customer Accounts | `extensions/customer-account-asset-portal/shopify.extension.toml` |
| Step 6 fails to create test buyer | `EMAIL_FROM` env var missing on Worker → can't derive catch-all domain | `apps/worker/wrangler.toml` + `routes/admin-onboarding.ts` |
| Magic-link email never arrives | dev store on an unverified custom sender domain | Shopify admin → Settings → Notifications |
| Onboarding banner sticks at `pending` | `settings_json.onboarding.status` not flipped | `lib/onboarding-store.ts::markFinished` |
