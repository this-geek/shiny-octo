# Manual Test Checklist — 2026-05-25

Covers the manual steps deferred from the last two merged PRs:

- **#13** — `fix(extensions): make app embeds discoverable + move asset portal to Customer Account UI`
- **#14** — `feat(phase-1i): merchant onboarding wizard + magic-link welcome`

Steps are ordered so the test buyer created by the onboarding wizard (§3)
becomes the approved B2B buyer used to verify the asset portal (§5). Don't
re-run a section unless something earlier in the chain changed.

### 7a. Pre-application landing block

- [ ] In the theme editor → **App embeds**, a new entry **B2B Wholesale
      Landing** appears alongside the two from §5.
- [ ] Enable it. Set `landing_path` to (or keep) `/pages/wholesale`.
- [ ] Create a Shopify page at that path (any title; blank content is fine

- [ ] Visit the storefront `/pages/wholesale`. The block renders: heading,

      wholesale account** CTA that links to `/pages/wholesale-apply`.
- [ ] Edit the block's settings (heading, bullets — one per line, CTA
      label / path, footnote). Reload — changes reflect immediately.
- [ ] Visit any other storefront URL (`/`, a product page). The block
      renders **nothing** — it's safe to leave the embed enabled globally.

### 7b. Activation-nudge cron

The cron is `0 9 * * *` UTC. Two ways to verify without waiting a day:

- [ ] **Wrangler-driven test:**
      `wrangler dev --test-scheduled` then `curl
      'http://localhost:8787/cdn-cgi/handler/scheduled?cron=*+*+*+*+*'`.
      Worker logs should print `activation-nudges: scan complete` with
      `scanned: N, sent: M`.
- [ ] Or use the production cron: confirm in the Cloudflare dashboard →
      Workers → b2b-companion → Triggers that the cron is listed and was
      last invoked at the most recent 09:00 UTC.

Functional check (independent of when the cron last ran):

- [ ] Pick an approved application whose `decided_at` is roughly 14
      (or 30 / 60) days ago. If none exists, manually
      `UPDATE applications SET decided_at = strftime('%s','now') - 14*86400
       WHERE id = <id> AND status = 'approved'` in D1.
- [ ] Confirm the buyer has placed **no** orders since approval (or the
      probe will skip them by design).
- [ ] Trigger the cron (or wait for 09:00 UTC). The buyer's inbox
      receives the *Your wholesale account is ready when you are* email
      (14-day) or the 30 / 60 day variant.
- [ ] `select * from application_nudges where application_id = <id>` in
      D1 shows the milestone row.
- [ ] Re-run the cron the same day. No duplicate email — the second run
      hits the `INSERT OR IGNORE` on the milestone row.
- [ ] Now place a real order as the buyer (any product). Move the
      application's `decided_at` to 30 days ago. Re-run the cron. The
      30-day nudge does **not** send — the order probe filtered it out.

## Quick triage matrix

| Symptom | Likely cause | Where to look |
|---|---|---|
| App embeds don't show up in theme editor | `shopify app deploy` didn't run, or wrong theme | `extensions/theme-app-extension/shopify.extension.toml` |
| Customer-account asset portal page absent | extension not enabled in admin → Customer Accounts | `extensions/customer-account-asset-portal/shopify.extension.toml` |
| Step 6 fails to create test buyer | `EMAIL_FROM` env var missing on Worker → can't derive catch-all domain | `apps/worker/wrangler.toml` + `routes/admin-onboarding.ts` |
| Magic-link email never arrives | dev store on an unverified custom sender domain | Shopify admin → Settings → Notifications |
| Onboarding banner sticks at `pending` | `settings_json.onboarding.status` not flipped | `lib/onboarding-store.ts::markFinished` |
| **B2B Wholesale Landing** block missing in App embeds | `shopify app deploy` didn't pick up the new `b2b-preapplication.liquid` | `extensions/theme-app-extension/blocks/` |
| Cron never runs | `[[triggers]]` not present in deployed `wrangler.toml` | `apps/worker/wrangler.toml` |
| Nudges sent twice to the same buyer | `application_nudges` row missing (0004 migration didn't run) | `migrations/0004_phase1j_nudges.sql` |
| Tour banner re-appears every page load | KV write failing — check `KV_SESSIONS` binding | `lib/tour-state.ts` |
| Profile tab blank | Worker can't reach Shopify Admin GraphQL — token decrypt or scope issue | `lib/company-profile.ts` + scope list |
