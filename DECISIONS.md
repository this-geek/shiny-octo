# Decision Log

Resolves the ambiguities raised against `b2b-app-requirements.md` v0.1. Each
entry is a working decision; revisit only when the listed trigger fires.

| # | Topic | Decision | Trigger to revisit |
|---|---|---|---|
| 1 | Repo shape | pnpm monorepo scaffolded by Shopify CLI. `apps/worker` (Hono on Workers, hosts API + App Proxy), `apps/admin` (Polaris + App Bridge 4 + Remix), `extensions/theme-app-extension`, `extensions/functions/{cart-transform,cart-validation,delivery-customization}`, `packages/shared` for cross-target pricing logic. | Worker bundle size > 1MB compressed, or build time > 90s. |
| 2 | Image processing | Cloudflare Images for variants (original/1200/300). Simpler API, native variants, predictable cost. | Monthly image bill exceeds 2× equivalent R2 + Transforms cost at projected volume. |
| 3 | R2 bucket layout | Single shared bucket, keys prefixed `shops/<shop_id>/<entity>/<id>/<variant>`. | A tenant contractually requires physical isolation, or R2 list operations become a bottleneck. |
| 4 | D1 partitioning | Single logical DB with `shop_id` column; no per-shop DBs at launch. | Any single shop exceeds 5M rows in `asset_downloads`, or query p95 > 50ms. |
| 5 | Catalogs ↔ Markets ↔ Tiers | Catalog assigned to the Company Location provides base price. Markets contributes currency/locale/tax only. Our `cart-transform` Function discounts off whatever line price Shopify passes in. Requirements doc §4.3 to be amended. | Shopify changes the Catalog-Location binding model. |
| 6 | Storefront gating | Liquid server-rendered hide of B2B-only PDPs (returns 404 in theme template) when `customer.b2b?` is false. Collections rely on Search & Discovery metafield filter excluding `b2b.b2b_only == true`. Theme App Embed Block performs client-side tier-price refinement only. Drop the "redirect to alternate collection" idea entirely. | Shopify ships a first-class B2B-only product flag. |
| 7 | Buyer auth | Customer Account API magic-link login only. No password-set flow. §7 Step 4 to be amended. | Shopify deprecates magic-link in Customer Accounts. |
| 8 | Application document uploads | Worker issues signed R2 PUT URLs; browser uploads directly to R2; form POST contains R2 keys + metadata only. Avoids App Proxy body-size and timeout limits. | Cloudflare changes R2 signed-PUT semantics. |
| 9 | App Proxy prefix | Read from `shops.settings_json.app_proxy.{prefix,subpath}`. No hardcoded `/apps/b2b-companion/...` paths in code or copy. | Shopify removes per-app proxy prefix config. |
| 10 | Tier cache in localStorage | UX cache only — used for instant price display on subsequent page loads. Functions and any server route that gates access **must** re-resolve from Company metafield / D1 on every request. Documented in the storefront block module header. | n/a — invariant. |
| 11 | Webhook list | Add `app/scopes_update`. Defer `app_subscriptions/update` until Phase 5 billing wiring. | App Store review flags additional required topics. |
| 12 | Pilot merchant constants | NZ-based merchant on Shopify Advanced, Dawn-based theme, ~20 customers tagged `wholesale`. At this volume the §6 Step 2 migration wizard is *useful but not critical-path* — a manual one-off import is viable, so the wizard moves to a P1 polish item rather than a Phase 1 P0. Tax-ID validators prioritise NZ IRD/GST format; AU ABN deferred. | Pilot merchant differs materially, or wholesale customer count grows past ~100 before launch. |
| 13 | Day 2 ordering | 1. Quick Order / CSV upload · 2. Sales Rep Portal & Impersonation · 3. PO numbers & order notes · 4. Saved lists & one-click reorder · 5. Credit limit enforcement · 6. Quote / RFQ · 7. Multi-user company management · 8. Order approval workflows · 9. Buyer-side order editing · 10. Watermarked downloads · 11. Co-op marketing credit · 12. Catalog import/export. Reordered to front-load highest-ROI items. | First 5 paying customers' requests differ from this order. |
| 14 | Asset portal bandwidth caps | Single fair-use ceiling of 250 GB/shop/month during pilot. Plan-tiered caps land with the public pricing model in Phase 5. | Any pilot tenant projected to exceed 250 GB/month. |
| 15 | Test buyer email | `test-buyer+<shop-handle>@<our-domain>` via a catch-all on our verified sending domain. | Email provider rejects plus-addressed catch-alls. |
